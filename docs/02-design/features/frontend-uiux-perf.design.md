# [Design] 프론트엔드 UI/UX 완성도 및 성능(속도) 최적화

## 1. 아키텍처 및 폴더 구조 (변경 대상)

수정 및 개선이 필요한 주요 파일들은 다음과 같습니다.

```text
client/
├── src/
│   ├── App.jsx             // 메인 레이아웃 및 뷰포트 관리, 메인 로고 폰트 (자간 수정)
│   ├── App.css             // 글로벌 스타일, 마진/패딩/글씨체 조정
│   ├── index.css           // 테일윈드/글로벌 스타일 
│   ├── components/         // (예상 경로 - 분석 후 정확한 파일 특정)
│   │   ├── Sidebar.jsx     // 사이드바 레이아웃 (길이 제한 및 스크롤 복구)
│   │   ├── FloatingSheet.jsx // 모바일 바텀시트
│   │   ├── ResultCard.jsx  // 마케팅 브랜딩 ('복합 통근 피로도 모델') 문구 추가
│   │   └── Footer.jsx      // 커피 아이콘 배경 높이 조절
```

## 2. UI/UX 디자인 수정 계획

### 2.1 네이밍 및 타이포그래피 (App.jsx, ResultCard.jsx 등)
- **네이밍 리뉴얼**: 
  - `워라밸 구출하기` -> `출퇴근 최적화 분석` (또는 `스마트 주거 탐색`)
  - `서바이브 엔진` -> `복합 통근 피로도 모델` (Comprehensive Commute Fatigue Model)
- **로고 자간 교정**: `App.css` 또는 `index.css`에서 `Search House` 텍스트를 감싸는 클래스에 `letter-spacing: 0.05em;` (또는 시각적으로 적절한 값) 적용.

### 2.2 디테일 교정 및 레이아웃 
- **커피 아이콘 컴포넌트**: 푸터나 하단 스폰서/도네이션 영역의 아이콘 래퍼의 `height` 또는 `padding-bottom`을 늘려 배경이 잘리는 현상 해결.
- **반응형 겹침 문제**: Flex/Grid 컨테이너의 `gap` 속성 및 `@media` 쿼리 분기점을 점검하여 컴포넌트들이 겹치는 현상 해결.

### 2.3 모바일 특화 UX 개선 (지도 몰입감 강화)
- **지도 중심 풀스크린 뷰**: 모바일(`@media (max-width: 768px)`)에서는 지도가 화면 100%(`100vh`)를 차지하도록 하고, 컨트롤러 패널을 최소화.
- **다이내믹 바텀 시트 구현**: 모바일 환경에서는 사이드바 대신 바텀 시트(Bottom Sheet) 형태의 레이아웃을 도입하여 초기에는 요약 정보만 노출하고, 스와이프 업/클릭 시 전체 화면으로 확장(`transform: translateY(...)` 기반)되도록 구성.

### 2.4 마케팅 브랜딩 요소 추가 (아실/호갱노노 벤치마킹 반영)
- **복합 모델 개념 노출**: 결과 페이지 또는 로딩 스크린에 **"단순 거리가 아닌 환승, 도보, 혼잡도를 모두 고려한 복합 통근 피로도 모델로 분석 중입니다."** 형태의 문구 노출.
- **직관적 가치 제안(Value Proposition)**: 결과 요약 시 '절약된 시간/체력'을 시각적으로 강조하는 툴팁 추가.
- **데이터 시각화 차별화**:
  - 지도 위에 아파트의 '히든 코스트(숨겨진 피로도 비용)'를 직관적인 색상(예: 신호등 컬러)이나 배지로 오버레이하여 표시 (호갱노노의 데이터 레이어 방식 차용).
  - 사이드바/바텀시트의 결과 리스트에 '최적 워라밸', '가성비 甲' 등의 직관적인 랭킹 태그를 부여하여 사용자의 빠른 의사결정 지원 (아실의 랭킹 시스템 차용).
- **네이버 부동산 다이렉트 연동**: `ResultCard.jsx` 컴포넌트 내에 "실제 매물 확인하기(네이버 부동산)" 버튼을 추가하고, 클릭 시 해당 아파트 단지명(또는 좌표)을 기반으로 네이버 부동산 URL을 동적으로 생성하여 새 창(`target="_blank"`)으로 연결.

### 2.5 사이드바 구조 복구
- 사이드바 컨테이너(`Sidebar.jsx` 또는 `App.jsx` 내 사이드바 영역)의 인라인 스타일 또는 클래스 중 강제로 높이를 제한하는 요소(`max-h-[..]`, `overflow-hidden` 등) 제거.
- 컨테이너에 `flex-1`, `overflow-y-auto`를 부여하여 내부 컴포넌트(스캔 버튼 아래쪽 결과)가 온전히 스크롤되도록 보장.

## 3. 성능 최적화 계획 (React)

- **렌더링 최적화**: `React.memo` 또는 `useMemo`, `useCallback`을 활용하여 지도 컴포넌트와 사이드바 UI 상태 분리 (불필요한 리렌더링 방지).
- **로딩 최적화**: 필요하다면 무거운 컴포넌트나 외부 라이브러리(차트, 특정 모달 등)에 `React.lazy`와 `Suspense` 적용. (초기 로딩 속도 TTI 지표 개선)

## 4. 작업 순서 (Do Phase)
1. **분석**: 전체 구조 및 CSS 클래스 탐색 (`App.jsx`, `App.css` 등).
2. **UI 디테일 수정**: 텍스트, 자간, 아이콘 배경, 사이드바 스크롤 교정.
3. **마케팅 문구 반영**: '복합 통근 피로도 모델' 개념 UI에 이식.
4. **성능 점검 및 최적화**: 렌더링 병목 지점 리팩토링.
5. **검증**: 모바일/데스크톱 뷰포트 교차 테스트.

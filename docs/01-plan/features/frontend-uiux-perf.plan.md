# [Plan] 프론트엔드 UI/UX 완성도 및 성능(속도) 최적화

## 1. 개요
- **기능명**: 프론트엔드 UI/UX 사용성 개선 및 성능 최적화
- **목적**: 전반적인 디자인 완성도(마진, 패딩, 크기, 뷰포트 비율 등)를 높이고, 사용성을 저해하는 레이아웃 잘림 현상과 어색한 네이밍을 수정하며, 로딩 및 렌더링 속도를 최적화합니다.
- **대상 범위**: 프론트엔드 웹/모바일 반응형 전체 UI

## 2. 상세 요구사항

### 2.1 디자인 및 UI/UX 디테일 개선 (완성도)
- **네이밍 리뉴얼**: 직관성이 떨어지는 "워라밸 구출하기", "서바이브 엔진" 등의 문구를 세련되고 명확한 이름으로 변경.
- **로고/타이포그래피 교정**: "Search House" 영문 텍스트에서 'h' 알파벳끼리 너무 가깝게 붙어있는 자간(letter-spacing) 문제 해결.
- **아이콘 컴포넌트 수정**: 하단 커피 아이콘의 배경 높이가 짧아 어색한 현상 수정.
- **레이아웃 겹침 및 비율 조정**: 마진, 패딩 값을 교정하여 화면 비율이 달라지거나 모바일 기기에서 요소들이 겹치거나 깨지는 현상 방지.

### 2.2 마케팅 및 모델 브랜딩 강화 (프롭테크 인사이트 반영)
- **복합 평가 모델 개념 소개**: 단순 물리적 거리(Distance)만을 측정하는 것이 아니라, 대중교통 환승, 도보 이동, 혼잡도 등 실제 체감 피로도를 반영한 **'복합 통근 피로도 모델'**을 사용한다는 개념을 UI 상에 자연스럽게 노출.
- **가치 제안(Value Proposition) 시각화**: 사용자가 결과를 보았을 때 단순한 부동산 검색이 아닌, '삶의 질(QoL)'을 분석해 준다는 느낌을 받을 수 있도록 결과 요약 컴포넌트나 툴팁(Tooltip), 인포그래픽 요소 추가.
- **데이터 시각화 차별화 (아실/호갱노노 벤치마킹)**:
  - 호갱노노의 '일조량 시뮬레이션', '경사도 레이어'처럼, 우리 서비스만의 강력한 무기인 **'히든 코스트(숨겨진 통근 피로도)'를 지도 위에 시각적인 오버레이나 직관적인 색상/아이콘으로 표현**.
  - 아실의 '데이터 랭킹화'처럼, 검색 결과 내에서 '가장 통근이 쾌적한 곳', '가성비가 가장 좋은 곳' 등을 한눈에 파악할 수 있는 직관적인 UI(배지, 랭킹 태그 등) 적용.
- **네이버 부동산 실매물 다이렉트 연동 (아웃링크)**: 분석된 결과 단지를 클릭할 경우, 해당 아파트의 실제 매물(전월세/매매)을 바로 확인할 수 있도록 '네이버 부동산 단지 페이지'로 바로 넘어가는 아웃링크 버튼/UX 추가. (실시간 매물 데이터 확보 한계 극복 전략)

### 2.3 모바일 특화 UX 개선 (지도 몰입감 강화)
- **지도 중심의 풀스크린 경험**: 네이버 지도나 카카오맵처럼 모바일 환경(세로형 뷰포트)에서는 지도가 화면의 대부분을 차지하여 시각적 몰입감을 극대화해야 함.
- **다이내믹 바텀 시트(Dynamic Bottom Sheet)**: 검색 폼이나 결과 리스트는 하단 바텀 시트로 배치하여, 사용자가 스와이프업(Swipe-up) 할 때만 확장되고 기본 상태에서는 지도를 가리지 않도록 축소(Minimzed)되게 처리.

### 2.4 사이드바 구조 및 사용성 개선
- **사이드바 길이 제한 해제 및 스크롤 복구**: 사이드바 길이가 임의로 축소되어 하단 영역이 보이지 않는 문제 해결.
- **컴포넌트 잘림 현상 해결**: "Scan(스캔)" 버튼 아래의 결과 및 추가 컴포넌트들이 잘리지 않고 온전히 접근 가능하도록 레이아웃 구조 개선 (필요시 내부 스크롤바 적용).

### 2.5 성능 최적화 (속도 기반)
- **렌더링 및 로딩 속도 향상**: 불필요한 리렌더링 방지 및 에셋/컴포넌트 로딩 속도 최적화. (Lighthouse 등 성능 지표 개선 목표)

## 3. 성공 기준 (Done Criteria)
- [ ] 모바일과 데스크톱 화면 비율 변경 시 컴포넌트 겹침이나 UI 깨짐 현상이 발생하지 않는다.
- [ ] 사이드바의 스캔 버튼 하단 내용이 모두 정상적으로 노출되며 스크롤이 가능하다.
- [ ] 변경된 네이밍과 로고 자간이 자연스럽게 적용되었다.
- [ ] 초기 렌더링 및 인터랙션 속도가 체감될 정도로 향상되거나, 개발자 도구 성능 지표가 개선되었다.

## 4. 고려/제약 사항
- 모바일(Bottom Sheet/Floating UI)과 데스크톱(Sidebar) 환경을 동시에 교차 검토해야 함.
- 기존에 구현된 지도 기능과 핵심 연산 로직(워라밸 계산 엔진 등)이 손상되지 않도록 UI 계층과 로직을 분리하여 작업.

# [Plan] 모바일 UX 획기적 개선 — 네이티브 앱 수준의 인터랙션 도입

## 1. 개요 (Overview)
모바일 환경에서 검색 가시성 저하 및 조작 불편 문제를 근본적으로 해결하기 위해, 입력 시 화면 레이아웃이 유연하게 반응하는 '모바일 포커스 모드'와 네이티브 앱 스타일의 UI를 구현함.

## 2. 주요 개선 사항 (Action Items)

### 2.1 검색 집중 모드 (Search Focus Mode)
- **트리거:** StationSearch 입력창 포커스 시 활성화.
- **레이아웃 변환:** 
    - 바텀 시트 즉시 `h-[100dvh]` (전체 화면) 확장.
    - 로고 및 기타 입력 필드 투명도 조절 또는 숨김 처리.
    - 입력창을 화면 최상단으로 이동 (Sticky).
- **공간 확보:** 키보드 영역을 제외한 나머지 모든 공간을 검색 결과 리스트에 할당.

### 2.2 Z-Index 및 레이어 정돈
- 역 검색 드롭다운의 `z-index`를 최상위(`z-[9999]`)로 고정.
- 바텀 시트 내부의 `overflow` 속성을 조정하여 리스트가 잘리는 현상 방지.

### 2.3 터치 및 피드백 강화
- 모든 버튼에 `active:scale-95` 및 `transition` 효과 강화.
- 입력창 클리어(X) 버튼 추가로 조작 편의성 향상.

## 3. 기술적 해결 방안
- `vh` 단위 대신 `dvh` (Dynamic Viewport Height)를 사용하여 키보드 대응.
- `framer-motion` 스타일의 부드러운 레이아웃 전환 애니메이션 적용 (Tailwind transition 활용).

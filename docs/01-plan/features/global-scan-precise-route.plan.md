# [Plan] 전역 스캐닝 및 리얼 타임(08:00) 정밀 분석 도입

## 1. 개요 (Overview)
수도권 전역의 아파트 단지를 대상으로 사용자의 조건에 맞는 최적의 입지를 스캐닝하고, 네이버 길찾기 API와 캐싱 시스템을 통해 08:00 도착 기준의 정밀한 통근 비용을 산출함.

## 2. 주요 작업 내용 (Action Items)

### 2.1 백엔드 알고리즘 개편 (Global Scanner)
- **전역 필터링:** 특정 역 주변이 아닌, `rent_transactions` 테이블 전체에서 가격/면적/준공년도 조건에 맞는 단지 상위 100개를 일차적으로 추출.
- **정밀 경로 분석:** 추출된 100개 후보와 직장 좌표 간의 실제 경로(자차/대중교통)를 네이버 API로 분석.
- **비용 절감 캐싱:** 동일 단지-직장 쌍의 경로 데이터는 `commute_cache` 테이블에 저장하여 중복 API 호출 방지.

### 2.2 네이버 API 연동 (Precise Route)
- 08:00 도착을 타겟으로 한 예측 교통 정보 활용.
- `server/lib/naver_api.py` 모듈 신규 개발.

### 2.3 프론트엔드 로딩 UX 고도화 (Premium Loading)
- **애니메이션:** Lottie 느낌의 세련된 로딩 애니메이션 또는 단계별 진행 상태 바 구현.
- **메시징:** "수도권 3만 개 단지 스캐닝 중...", "08:00 기준 실시간 교통정보 분석 중..." 등 단계별 피드백 제공.

## 3. 비용 및 성능 목표
- **API 비용:** 캐시 히트율 80% 이상 달성을 통해 월 무료 쿼터 내 운영.
- **응답 속도:** 캐시 존재 시 1초 이내, API 호출 시 최대 5초 이내 완료 및 로딩 UI로 체감 대기시간 단축.

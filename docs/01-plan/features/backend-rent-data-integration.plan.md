# [Plan] 백엔드 전월세 실거래가 연동 및 전월세 분리 최적화

## 1. 개요
- **기능명**: 국토교통부 아파트 전월세 실거래가 API 연동 및 전월세(전세/월세/반전세) 분리 처리 로직 구현
- **목적**: 기존 매매 실거래가 기반의 분석에서 벗어나, 실제 주거 탐색 유저들이 가장 필요로 하는 '전월세' 데이터 기반으로 워라밸 기회비용을 계산합니다. 특히 전세와 월세, 그리고 섞여있는 '반전세'를 명확한 기준으로 통일/분리하여 유저에게 제공합니다.
- **대상 범위**: 백엔드 데이터 수집기(`collector_rent.py` 신설), 분석 API(`main.py`), 프론트엔드 UI(`App.jsx`)

## 2. 상세 요구사항

### 2.1 백엔드 데이터 수집 파이프라인 신설
- **API 연동**: 제공된 서비스 키와 국토교통부 **아파트 전월세 실거래가 자료 API**(`http://apis.data.go.kr/1613000/RTMSDataSvcAptRent`)를 사용하여 데이터 수집.
- **기존 수집기 유지**: 기존 매매 데이터 수집 로직(`collector.py`)은 그대로 보존하고, 전월세 전용 수집 스크립트(`collector_rent.py`)를 분리하여 운영.
- **데이터베이스 구축**: `search_house.db`에 전월세 전용 테이블(`rent_transactions`)을 생성하여 보증금(`deposit`)과 월세(`monthly_rent`)를 각각 Integer 타입으로 분리하여 저장.

### 2.2 전세 / 월세 / 반전세 분리 및 환산 알고리즘 (핵심)
실제 데이터에는 전세(월세 0), 월세(보증금 소액+월세), 반전세(보증금 고액+월세)가 혼재되어 있습니다. 랭킹을 매기기 위해 이를 **'월 주거비용'**으로 통합 환산하는 로직을 적용합니다.

- **전월세 전환율 적용 (법정/시장 전환율 평균 약 5~5.5% 가정)**:
  - 기회비용 계산 시 보증금에 대한 이자 비용을 연 4%로 계산하여 월 비용으로 산출.
  - **월 주거비용(Housing Cost)** = `(보증금 * 0.04 / 12) + 실제 월세`
- 데이터베이스 쿼리 시, 보증금과 월세 유무에 따라 데이터를 태깅:
  - `월세가 0`이면 ➡️ **[전세]**
  - `월세가 0보다 크면` ➡️ **[월세/반전세]**
- 프론트엔드에 전달할 때는 단지별로 '평균 전세 보증금', '평균 월세(보증금/월세)'를 분리해서 넘겨주어 화면에 명확히 표기.

### 2.3 프론트엔드 UI 반영
- **매매 기능 임시 숨김**: 사용자가 혼란을 겪지 않도록 기존의 "매매 / 임대" 토글을 숨기고(혹은 임대 전용으로 고정), 무조건 전월세 기반 분석 결과만 노출.
- **UI 텍스트 개편**: 
  - "추정 전세가" -> **"평균 전세"** 또는 **"평균 월세"**
  - 가격 표기 시: 전세인 경우 `전세 O억 O천`, 반전세/월세인 경우 `월세 O천 / OO만` 형태로 직관적으로 렌더링.

## 3. 성공 기준 (Done Criteria)
- [ ] `collector_rent.py`가 정상 작동하여 DB에 보증금과 월세가 분리되어 저장된다.
- [ ] `main.py` 분석 엔진이 반전세를 포함한 보증금+월세 데이터를 기반으로 '월 주거비용'을 통합 계산하여 최적 입지를 산출한다.
- [ ] 프론트엔드에서 매매 옵션이 사라지고, 결과 카드에 전세/월세 여부와 보증금/월세 금액이 명확히 나누어 표기된다.

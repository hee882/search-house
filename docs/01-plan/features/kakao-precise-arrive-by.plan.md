# [Plan] 카카오 모빌리티 정밀 시간 산출 (Arrive-by 08:00)

## Executive Summary

| 항목 | 내용 |
| ---- | ---- |
| Feature | kakao-precise-arrive-by |
| 시작일 | 2026-03-16 |
| 목표 | 카카오 API의 '미래 운행 정보'를 활용하여 08:00 도착 시점과 18:00 출발 시점의 실제 교통 데이터를 기반으로 정밀 시간 산출 |

### Value Delivered

| 관점 | 내용 |
| ---- | ---- |
| **Problem** | 카카오 API는 '출발 시간' 기준만 지원하여 사용자가 원하는 '08시 정각 도착' 시점의 데이터를 얻기 어려움 |
| **Solution** | 도착 희망 시간(08:00)을 기준으로 출발 시간을 역산하는 'Binary Search simulation' 알고리즘 도입 |
| **Function UX Effect** | "08:00 정각에 회사에 도착하기 위해 필요한 실제 주행 시간"을 정확히 노출 |
| **Core Value** | 단순 보정치가 아닌 실제 카카오 빅데이터 기반의 교통 정보 제공으로 서비스 신뢰도 완성 |

## 1. 정밀 알고리즘 설계

### 1.1 "08:00 도착" 시뮬레이션 (Simulation)
API가 직접적인 Arrive-by를 지원하지 않으므로, 백엔드에서 아래 과정을 수행합니다.
1.  **초기값:** 08:00 - (직선거리 예상 시간) 시점을 첫 출발 시간으로 설정.
2.  **호출:** 해당 시점의 `future/directions` 호출하여 `duration` 확인.
3.  **역산:** `출발 시간 + duration`이 08:00이 되도록 출발 시간을 앞뒤로 조정 (최대 2회 반복으로 최적값 근사).
4.  **확정:** 08:00에 가장 근접하게 도착하는 시점의 소요 시간을 최종값으로 사용.

### 1.2 "18:00 출발" 정밀화
- 퇴근은 명확히 18:00 출발로 고정하여 해당 시점의 미래 정체 데이터를 즉시 획득.

## 2. 구현 계획

### Phase 1: `kakao_api.py` 고도화
- `arrive_by` 파라미터 추가.
- 내부적으로 출발 시간을 조정하며 API를 호출하는 루프 로직 구현.
- API 쿼터 절약을 위해 캐싱 키에 `arrive_by` 여부 포함.

### Phase 2: `main.py` 연동
- 출근 호출 시 `arrive_by="0800"` 옵션 부여.
- 퇴근 호출 시 `departure_time="1800"` 옵션 부여.

## 3. 검증 방법
- 특정 경로(예: 용인 -> 판교)에 대해 08:00 도착을 위한 출발 시각이 교통 상황에 따라 유동적으로 계산되는지 확인.
- 임의의 보정치(1.35x 등)를 모두 제거하고 순수 API 결과값으로만 동작하는지 확인.
